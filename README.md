## Tareas Pendientes y Funcionalidades por Arreglar

### Reservas
- [ ] **Superposición de horarios:** Validar y bloquear la creación de reservas en una sala si los horarios chocan con una reserva existente.
- [ ] **Límite de hora de cierre:** Restringir la duración máxima permitida para garantizar que ninguna reserva finalice después de la hora de cierre del CRAI.
- [ ] **Restricción de fechas:** Limitar el selector del calendario para evitar la selección de fechas pasadas o fuera del límite de días permitidos.

### Persistencia de Datos y Sesión
- [ ] **Mantener sesión activa:** Configurar la aplicación para que recuerde al usuario al cerrarla y volverla a abrir, para evitar que se pida el inicio de sesión cada vez que se inicie la app, a menos que el usuario cierre su sesión de forma manual.

### Interfaz de Usuario (UI)
- [ ] **Paleta de colores en pantallas secundarias:** Arreglar diseño de ReservasScreen, PrivacyScreen y HelpSupportScreen para que utilicen la paleta de colores de la aplicación (azul UCA).
- [ ] **Datos dinámicos en pantalla de perfil:** Conectar la pantalla de perfil a la base de datos para mostrar las estadísticas reales del usuario (total de reservas y horas acumuladas).
